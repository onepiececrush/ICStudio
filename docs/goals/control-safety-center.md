# 控制命令安全中心

## 目标

为所有高危控制命令增加安全保护。

## 高危操作

包括：

- 启动
- 停止
- 复位
- 急停
- 有功功率给定
- 无功功率给定
- 参数批量写入
- 故障清除
- 固件升级
- 从机模拟故障注入

## 安全流程

执行前：

1. 检查用户权限
2. 检查当前连接设备
3. 检查设备运行状态
4. 检查参数范围
5. 检查是否处于自测模式
6. 弹出二次确认

执行中：

1. 发送写入命令
2. 等待响应
3. 回读确认
4. 判断是否生效

执行后：

1. 记录操作日志
2. 记录写入前后值
3. 显示执行结果
4. 失败时显示原因

## 操作日志

字段：

- 时间
- 用户
- 操作类型
- 设备
- 地址
- 写入值
- 写入前值
- 写入后值
- 结果
- 失败原因

## 验收标准

1. 启动/停止/复位必须二次确认。
2. 功率给定必须检查范围。
3. 写入后必须回读校验。
4. 失败原因必须明确。
5. 所有操作写入日志。
6. 自测模式和真实设备模式有明显区分。
