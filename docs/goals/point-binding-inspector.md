# 点位绑定追踪器

## 目标

实现一个点位追踪功能。用户点击页面上的任意数据值，可以查看该值从通信报文到页面显示的完整链路。

## 需要追踪的信息

- 页面组件 ID
- 页面名称
- 设备实例
- 协议版本
- 寄存器地址
- 功能码
- 数据类型
- 字节序
- 倍率
- 偏移
- 单位
- 原始寄存器值
- 解码后的工程值
- 页面格式化显示值
- 最后更新时间
- 最后一次请求报文
- 最后一次响应报文
- 通信耗时
- 错误信息

## UI 设计

新增右侧抽屉面板：点位详情。

用户点击首页 KPI、拓扑节点、PCS 卡片、BMS 卡片、表格单元格时，右侧展开。

面板分区：

1. 基本信息
2. 协议信息
3. 当前值
4. 报文来源
5. 最近变化
6. 错误诊断

## 示例

点击首页“总有功功率”：

- 页面组件：HomeTotalActivePowerCard
- 地址：14006
- 名称：总有功功率
- 类型：int16
- 单位：0.01kW
- 原始值：125000
- 工程值：1250.00
- 页面显示：1,250.00 kW
- 最后更新时间：14:35:40
- 响应耗时：8.2ms

## 错误提示

如果页面显示错误，要能提示：

- 无绑定点位
- 点位未读取
- 协议倍率缺失
- 类型解析失败
- 通信超时
- 值越限
- 单位不一致

## 验收标准

1. 首页所有核心 KPI 都能点击追踪。
2. PCS 模块矩阵每个卡片都能追踪。
3. BMS、液冷、动环、电表、箱变卡片都能追踪。
4. 能显示最近一次请求和响应报文。
5. 通信失败时能显示失败原因。
6. 自测模式下可以对比模拟器期望值。
