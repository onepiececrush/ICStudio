# 首页自测模拟闭环 Goal

## 目标

实现“首页自测模拟闭环”功能。软件在没有真实 PCS/BMS 设备时，可以启动内置 Modbus TCP 从机模拟器，然后首页作为主站连接该模拟器读取数据，并自动校验首页显示是否正确。

必须是真实通信闭环：

内置 Modbus TCP Server  
→ 模拟寄存器数据  
→ 首页通过 Modbus TCP 主站读取  
→ 首页按协议解析并显示  
→ 校验器对比模拟期望值和首页实际值

禁止绕过通信层直接给首页塞假数据。

## 自测模式

新增 Loopback Simulation Mode。

点击“启动首页自测模拟”后：

1. 启动内置 Modbus TCP Server。
2. 默认监听 `127.0.0.1:1502`。
3. 默认 Unit ID 为 `1`。
4. 自动加载 PCS/BMS 协议模型。
5. 生成确定性的模拟数据。
6. 自动把首页主站连接切换到 `127.0.0.1:1502`。
7. 首页按正常轮询流程读取数据。
8. 校验器对比模拟器值和首页显示值。

顶部必须显示“当前为自测模式”，避免用户误以为连接真实设备。

## MVP 通信要求

第一版必须支持：

- Modbus TCP Server
- 功能码 03：读 Holding Register
- 功能码 04：读 Input Register，如首页需要
- 功能码 06：写单寄存器
- 功能码 16：写多个寄存器
- 监听地址：127.0.0.1
- 监听端口：1502
- Unit ID：1

RTU Slave 和完整四区可放第二阶段，但架构要预留。

### PCS 模块

模拟 16 台 PCS。

基地址规则：

- PCS1 base = 15001
- PCS2 base = 15001 + 500
- PCS3 base = 15001 + 1000
- PCS16 base = 15001 + 15 * 500

每台至少模拟：

- base + 9：运行状态
- base + 10 ~ base + 19：模块温度
- base + 20 ~ base + 29：故障信息
- base + 50 ~ base + 84：电压、电流、功率类数据

默认状态：

- PCS1、PCS2、PCS4、PCS5、PCS6、PCS8、PCS9、PCS10、PCS12、PCS13、PCS14、PCS16：运行
- PCS7、PCS11：待机
- PCS3：故障
- PCS15：离线

PCS3 默认注入一个故障，用于验证首页告警中心。

## 协议转换要求

模拟器内部可以保存工程值，但对外响应必须是 Modbus 原始寄存器值。

需要实现：

- `encodeEngineeringValueToRaw(point, value)`
- `decodeRawToEngineeringValue(point, rawRegisters)`

流程必须是：

工程值  
→ 按协议 scale、offset、data_type、byte_order 编码成 raw  
→ Modbus 响应 raw  
→ 首页再按协议解析成工程值  
→ 页面显示  
→ 校验器对比

## 首页校验面板

新增“首页数据校验”面板。

字段：

- 页面组件
- 绑定地址
- 点位名称
- 模拟器期望值
- 首页解析值
- 页面显示值
- 单位
- 误差
- 校验结果

规则：

- 数值允许误差 0.01，或按点位精度判断。
- 枚举文本必须一致。
- bit 状态必须一致。
- 单位必须一致。
- 页面显示为空、NaN、-- 判定失败。
- 通信失败单独显示“通信失败”。

## 自测操作按钮

自测面板增加按钮：

1. 修改总有功功率  
   将 14006 从 1250.00 改为 1500.00，首页应在 1~2 个轮询周期内更新。

2. 修改 SOC  
   将 25609 从 78.50 改为 66.60，首页 BMS 卡片应更新。

3. 注入 PCS3 故障  
   PCS3 卡片变红，告警中心严重故障 +1。

4. 清除故障  
   PCS3 恢复运行，告警数量下降。

5. 模拟通信中断  
   停止 TCP Server 或让 Unit ID 不响应，首页通信状态变异常。

## 自动化测试

至少增加这些测试：

1. 内置从机启动测试  
   启动 127.0.0.1:1502，读取 14001，期望返回 12。

2. 首页 PCS 汇总读取测试  
   校验 14001、14002、14006、14007、14031、14032。

3. 首页 BMS 数据测试  
   校验 SOC、SOH、总压、总流、总功率。

4. PCS 模块矩阵测试  
   校验 16 台 PCS 卡片存在，PCS3 故障，PCS15 离线。

5. 故障注入测试  
   注入 PCS3 故障后，首页告警数量增加，PCS3 卡片变红。

6. 模拟值变化测试  
   修改 14006 后，首页显示值自动更新。

## 安全要求

自测模式不能破坏真实设备连接。

要求：

1. 如果当前已连接真实设备，启动自测前弹窗确认。
2. 退出自测后恢复原通信配置。
3. 自测配置不能覆盖用户真实工程配置，除非用户手动保存。
4. 页面必须明确显示“自测模式”。

## 验收标准

完成后必须满足：

1. 不连接真实设备，点击“启动首页自测模拟”，首页能显示完整数据。
2. 首页数据来自本机 Modbus TCP Server，不是假数据。
3. 修改 14006 后，总有功功率卡片自动更新。
4. 修改 25609 后，SOC 自动更新。
5. 注入 PCS3 故障后，PCS3 卡片和告警中心变化。
6. 校验面板逐项显示期望值、实际值、误差、通过/失败。
7. 停止模拟器后，首页显示通信异常。
8. 自测模式不覆盖真实设备连接配置。
9. 所有自测数据和校验结果有日志。
10. 模拟器、通信、首页、校验器解耦。
