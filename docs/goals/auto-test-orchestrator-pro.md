# 自动化测试编排器 Pro

## 目标

实现可配置的自动化测试编排器，不再只能运行写死的测试项。

## 测试步骤类型

支持：

- 连接设备
- 读取点位
- 写入点位
- 等待时间
- 等待条件
- 断言数值
- 断言枚举
- 断言 bit
- 注入故障
- 清除故障
- 启动场景
- 停止场景
- 检查报文
- 生成截图
- 导出报告

## 测试用例模型

TestCase：

- case_id
- case_name
- device_type
- tags
- steps
- timeout
- retry
- expected
- result
- logs

TestStep：

- step_id
- type
- target_device
- point_address
- value
- condition
- timeout
- on_fail

## UI

左侧：测试用例树  
中间：步骤编排器  
右侧：步骤属性  
底部：运行日志  
顶部：运行全部 / 运行选中 / 停止 / 导出报告

## 内置模板

- PMU 通信测试
- PCS 在线测试
- PCS 启动测试
- PCS 停止测试
- PCS 功率给定测试
- BMS 数据范围测试
- 液冷通信测试
- 动环急停测试
- 故障恢复测试
- 首页自测模拟闭环测试

## 报告

报告包含：

- 测试时间
- 工程名称
- 协议版本
- 设备信息
- 用例结果
- 步骤日志
- 失败原因
- 通信报文
- 截图
- CSV/HTML/PDF 导出

## 验收标准

1. 可以新建测试用例。
2. 可以配置读写点位步骤。
3. 可以设置断言。
4. 可以运行测试并显示结果。
5. 可以调用从机模拟器做故障注入。
6. 可以生成测试报告。
7. 现有自动化测试项可以迁移成模板。
