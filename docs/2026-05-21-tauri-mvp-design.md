# ICStudio Tauri MVP 设计

生成日期：2026-05-21

## 目标

基于现有上位机功能文档、通用平台规划、HTML 原型、效果图和 PCS Modbus V3.13 协议表，启动 ICStudio 的 Tauri 桌面应用开发。第一阶段先做成可运行、可扩展的通用工业上位机壳体和核心页面骨架，为后续通信、协议导入、实时监控、参数配置、从机模拟和自动化测试铺好边界。

## 技术选型

- 桌面壳：Tauri v2。
- 前端：React + TypeScript + Vite。
- 后端：Rust Tauri commands，先定义通信、协议、工程配置、运行状态的模块边界。
- UI：复用现有 `index.html` 和两张效果图的工业仪表盘方向，但工程化拆成组件和数据模型，避免继续维护单个大型 HTML 文件。
- 协议输入：第一阶段把 Excel 协议表作为外部资料保留，先建立导入和解析边界；完整解析器在后续迭代实现。

## 第一阶段范围

第一阶段只做能稳定启动和演示主流程的 MVP：

1. 在项目根目录创建 Tauri 应用工程文件，原始需求文档、原型、图片和协议文件统一归档到 `docs/source-materials/`。
2. 实现应用壳：左侧导航、顶部项目和连接状态、主工作区。
3. 实现首页总览：健康度、在线设备、告警、仿真、自动化测试、固件升级、储能拓扑、趋势卡片、设备状态表、最近活动。
4. 建立模块导航占位：通信中心、协议管理、设备管理、实时监控、参数配置、波形录波、告警管理、自动化测试、从机模拟、数据服务、报表中心、固件升级、系统设置。
5. Rust 后端提供最小 `get_app_snapshot` command，返回当前工程、连接、模块统计和活动数据。
6. 前端通过 Tauri invoke 获取初始状态；浏览器开发模式下使用同结构 mock fallback，方便普通 Vite 调试。

## 边界与后续

本轮不直接实现真实硬件通信、完整 Excel 协议解析、数据库、固件升级状态机或从机模拟协议栈。真实能力后续按以下顺序迭代：

1. 通信中心：Modbus TCP 主站、报文助手、日志和统计。
2. 协议管理：Excel/JSON 导入、字段映射、寄存器点表模型。
3. 实时监控和参数配置：按协议点表生成表格，支持读写。
4. 从机模拟：Modbus TCP Server、多 Unit ID、寄存器表、故障注入。
5. 自动化测试、波形录波、固件升级。

## 验收

- 项目根目录可安装依赖并运行前端构建。
- 首页在桌面宽度下呈现工业仪表盘效果，主要模块入口完整。
- Tauri 后端 command 可被前端调用或在浏览器模式下安全降级。
- 根目录原始文档、图片、协议表和 `index.html` 不被破坏。
