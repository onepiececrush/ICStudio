# PCS 上位机功能文档

生成日期：2026-05-20  
适用工程：`EPS520` / `PCS` 调试上位机  
当前工程版本：`10.06.6`，来自 `EPS520.pro` 中的 `VERSION` 定义  
文档依据：当前仓库源码、Qt UI 文件、协议配置、自动化测试文档与配置文件静态分析

## 1. 项目定位

本项目是 EPS520/PCS 一体机调试上位机，README 中描述为“EPS520项目的调试上位机的代码”。软件面向 PCS、PMU、BMS、液冷、动环、箱变测控等设备调试与运维场景，主要能力包括：

- Modbus RTU / Modbus TCP 通信连接、收发报文查看与手动报文发送。
- EMS/PMU 汇总视角下的 PCS、BMS、液冷、动环、箱变测控寄存器数据展示与配置。
- PCS 实时数据、运行控制、保护参数、标定参数、厂家功能和故障配置。
- 历史事件读取、故障名称映射、分页浏览与 CSV 导出。
- 实时波形、故障录波读取、波形通道选择、波形 CSV 导入导出与游标查看。
- 历史电量读取、柱状图展示、充放电类型与时间粒度切换。
- PCS 固件升级、PMU 分发升级、升级压力测试与版本校验。
- 脉冲调试工具。
- PMU/PCS/液冷/动环相关自动化测试。

## 4. 总体架构

主程序采用一个主窗口承载多个功能页面。`MainWindow` 管理菜单、通信连接、通信线程、页面生命周期和跨页面信号连接。业务页面本身不直接持有串口或 TCP 对象，而是通过信号把待发送 Modbus 报文交给 `serialThread`，再由主窗口把响应转发回当前页面。

核心链路如下：

```text
QApplication
  -> MainWindow
      -> serialThread，运行在 QThread 中
      -> 参数设置页面 parameter
      -> EMS 综合页面 EMS
      -> 故障录波页面 wave
      -> 历史事件页面 events
      -> 历史电量页面 power
      -> 固件升级页面 upgrade
      -> PMU 升级页面 PMUpgrade
      -> 调试工具 WinTest
      -> 自动化测试 AutoTestWidget
```

页面切换时，`MainWindow::win_close()` 会停止当前页面定时器、清理通信队列、关闭页面并断开该页面和通信线程之间的信号连接。进入新页面时再按页面类型重新连接信号槽。

## 6. 通信系统

通信系统集中在 `serialThread`。

### 6.1 通信模式

RTU 模式：

- 使用 `QSerialPort`。
- 数据位 8 位，停止位 1 位，无校验。
- 发送前自动追加 Modbus CRC16。
- 串口收到数据后使用短定时器聚合读取。

TCP 模式：

- 使用 `QTcpSocket`。
- 使用 `LowDelayOption`。
- 主窗口发送普通 Modbus PDU 时，由 `serialThread` 追加 MBAP 头。
- 收包时剥离 MBAP 头，再交给上层页面。

### 6.2 发送队列与心跳

通信线程维护发送队列：

- 当前可写时直接发送。
- 当前等待响应时，新请求进入 `tx_data_queue`。
- 响应到达后调用 `tx_data_check()` 继续发送下一包。
- 队列和循环发送为空时启动通信心跳。

心跳报文默认读寄存器 `0x36B2`，功能码 `0x03`，每 2 秒触发一次。主窗口用心跳和状态回读更新系统状态灯。

### 6.3 批量/阻塞读取

波形页面需要高速读取时，会使用 `serial_sendBlock_slot()`。TCP 模式下该函数直接在循环中写请求、等待响应、发出 `response`，并由主窗口的停止标志控制退出。RTU 模式下退化为普通循环发送。

### 6.4 响应分发

收到响应后，通信线程发出两个信号：

- `response(QByteArray)`：原始响应，给串口助手、波形、历史事件、历史电量等页面。
- `serial_para_signal(uint16_t addr, QByteArray data)`：根据当前请求推导寄存器起始地址，给 EMS、参数设置、升级、自动化测试等需要按地址处理的页面。

## 7. 协议、配置与文件 IO

### 7.1 协议来源

程序运行时将 `agreement_path` 设置为“可执行文件所在目录/agreement”。源码仓库中包含：

- `agreement/EMS/*.xlsx`：EMS/PMU/PCS Modbus 协议表，包含多个版本。
- `agreement/PCS/*.xlsx`：PCS 调试通讯协议和标定参数。
- `agreement/故障表/*.xlsx`：PCS 故障点位表。
- `agreement/config.json` 与 `agreement/config_sys.json`：页面配置、读写范围、PACK 偏移、导出表头、PMU 上传配置。

EMS 页面启动时会进入 `agreement/EMS`，默认取目录中的第一个文件作为协议表；如果未找到，会弹出文件选择框。

### 7.2 配置文件能力

`agreement/config.json` 当前包含：

- `Enable_quickwrite`：常用参数页面是否启用单元格直写。
- `Enable_SynchronizeTime`：是否启用自动授时相关配置。
- `Range`：关键功率给定寄存器的最小值/最大值限制，例如 `14055`、`14056`、`15405`、`15406`。
- `emsTabPage`：EMS 页面各 tab 是否显示。
- `flexview`：常用参数页面不同视图的读写寄存器范围。
- `PACK`：PCS、液冷等可索引设备的基地址、步长、数量与偏移类型。
- `other.ChannelType`：波形通道名称。
- `PmuUploader`：PMU 升级上传的用户名、密码、远端目录。
- `flexExportHeader`：常用参数实时 CSV 导出的字段顺序。

### 7.3 JSON 工具

`Utility::load_json_file()` 和 `Utility::save_json_file()` 提供 JSON 读写：

- 读取不存在或解析失败时返回空对象。
- 保存时会自动创建目标目录。
- 保存格式为 Qt 默认 JSON 文档格式。

### 7.4 Excel 与 CSV

项目中 Excel 主要用于协议表和故障表解析。CSV 主要用于：

- 历史事件导出。
- 波形数据导出和导入。
- 常用参数实时采集导出。

## 8. EMS 综合调试页面

`EMS` 是当前项目中功能最多的页面。它通过协议 Excel 解析寄存器，并动态生成多个表格视图。

### 8.1 数据模型

关键全局数据：

- `g_ems_modbus_info`：内存中的 EMS/PCS/BMS/动环/厂家功能综合数据结构。
- `g_mb_read_reg_info`：寄存器起始地址、长度、内存地址映射表。
- `addr_to_excel`：寄存器地址到 Excel 协议信息的映射。
- `ems_view_list`：页面表格到寄存器区间、显示字段、数据偏移的映射。
- `fault_excel`：故障 ID 到故障名称的映射。

EMS 收到 `0x03` 响应后，会：

1. 按起始地址写入 `g_ems_modbus_info` 对应内存。
2. 根据数据类型进行大小端转换。
3. 根据当前 tab 和 view model 刷新表格。
4. 对特殊页面执行额外处理，例如常用参数合并表、故障配置、导出采集。

### 8.2 PCS 实时数据

页面 tab：`PCS实时数据`  
主要视图：

- 汇总信息：由 `14001:14008;14009:14021;14022:14032;14033:14040` 合并生成。
- 模块信息：由 `15001:15010;15011:15020;15021;15051:15085;15201:15219` 合并生成。

功能点：

- 支持 1 到 16 台 PCS 模块选择。
- 模块寄存器按 `15001 + index * 500` 偏移。
- 自动定时读取当前页面数据。
- 点击表格单元格后，可将对应寄存器数据展示为 16 位 LED/二进制灯。

### 8.3 BMS 实时数据

页面 tab：`BMS实时数据`  
主要寄存器区间：

- `25601:25659`：BMS 簇级/系统相关信息。
- `34001`、`34053`、`34079`、`34080`：PACK 相关信息。
- `33601:33615`：BMS 故障信息。

功能点：

- 支持 PACK 选择。
- PACK 地址有固定偏移逻辑。
- 实时读取和表格展示。

### 8.4 故障数据

页面 tab：`故障数据`  
包含：

- PCS 故障表：`15021`
- BMS 故障表：`33601:33615`
- 动环故障表：`13203:13206`

故障表数据来自协议和故障 Excel，页面用于集中展示当前故障位。

### 8.5 PCS 参数

页面 tab：`PCS参数`  
主要区间：

- `14051:14064`：系统/PMU 调控相关。
- `15301:15306`：时间设置。
- `15307:15310`：RS485 设置。
- `15311:15335`：网口设置。
- `15401:15414`：PCS 控制相关。

功能点：

- 可读写 PCS/PMU 相关控制参数。
- 支持范围校验，关键范围由 `config.json` 的 `Range` 提供。
- 支持启动、停止、复位等快捷按钮。

快捷控制当前实现：

- 启动：先清零有功/无功给定 `15405`、`15406`，再写 `14051 = 1`。
- 停止：先清零有功/无功给定，再写 `14051 = 2`。
- 复位：写 `14052 = 1`。

### 8.6 BMS 参数与 BMS 标定

BMS 参数 tab：

- `45001:45046`：电池参数。
- `45048:45098`、`45099:45149`：故障保护参数。

BMS 标定 tab：

- `45199:45200`：均衡设置。
- `45201:45220`：VCMU 标定。
- 支持 VCMU 选择，选择 `all` 时发送 `0xFF`。

### 8.7 动环与液冷

页面 tab：`动环与液冷`  
主要寄存器：

- `13001:13021`、`13031`、`13052:13064`：液冷、箱体公共或相关状态。
- `13101:13146`：PCS 液冷机，支持液冷索引。
- `13201:13232`：动环 DI/DO、状态、告警等。
- `13300:13308`：电表/标定/阈值相关。
- `13501:13570`：箱变测控。

在某些读取流程中，页面会先写液冷选择寄存器再读对应液冷数据，实现多台液冷设备轮询。

### 8.8 常用参数

页面 tab：`常用参数`  
该页由 `config.json` 的 `flexview` 动态定义。当前默认视图包括：

- 常用参数
- PCS
- 液冷
- 箱变测控
- 厂家信息
- 故障信息

每个视图有独立的 `read` 和 `write` 地址段。页面左侧树支持按照 `PACK` 配置生成 PCS、液冷等子节点，点击节点后自动重建读写表格并按设备序号偏移地址。

功能点：

- 读表和写表并列显示。
- “全部读取”“全部写入”。
- `直写`：单元格变化时可直接写单个或对应寄存器。
- 读取常用参数后计算并显示：
  - 直流侧功率 = 电池直压 × 电池电流 / 1000
  - 充放电效率，根据直流功率与电网有功功率计算
- 支持实时导出常用参数到 CSV。

常用参数导出由 `FlexDataExporter` 实现：

- 开始导出后按 `flexExportHeader` 字段顺序写 CSV。
- 数据先进入内存队列。
- 每 5 秒或缓冲区满时刷新到磁盘。
- 停止导出时强制刷新剩余数据。

### 8.9 保护参数

页面 tab：`保护参数`  
寄存器基础区间：`40201:40280`  
页面支持四类保护参数切换：

### 8.10 参数标定

### 8.11 故障配置

页面 tab：`故障配置`  
核心区间：

- `40121:40184`：故障启用、自恢复、故障类型等配置。

## 9. 旧版 PCS 参数页面

`parameter` 页面是较早的固定点表 PCS 调试界面，仍在主窗口 PCS 菜单中保留。

它在头文件中定义了固定调试 Modbus 数据结构：

- 实时数据：`0x0000 ~ 0x00FF`
- 保护参数：`0x2000 ~ 0x20FF`
- 校准参数：`0x2100 ~ 0x21FF`
- 控制参数：`0x2200 ~ 0x22FF`
- 运行参数：`0x2300 ~ 0x23FF`

页面 tab 包括：

- 电网实时数据
- 保护参数
- 校准参数
- 控制参数

功能点：

- 读取实时数据、故障字、状态字。
- 读取/下发速断保护、系统侧保护、输出侧保护、直流侧参数。
- 读取/下发校准参数。
- 读取/下发控制参数和运行参数。
- 快捷控制启动、急停、复位、DC 预充、DC 主路、PWM、AC 旁路、AC 主路。

当前更通用和更活跃的寄存器动态视图集中在 `EMS` 页面。

## 10. 故障录波与实时波形

`wave` 页面负责示波器式波形显示和故障波形读取。

### 10.1 通道能力

默认创建 16 个 CH 通道，每个通道包含：

- 是否显示的复选框。
- 倍率/缩放值。
- 通道类型下拉框。
- 对应一条 QCustomPlot 曲线。

通道名称来自 `config.json` 的 `other.ChannelType`，默认包括：

- 逆变电压 a/b/c
- 电感电流 a/b/c
- 电网电压 a/b/c
- 输出电流 a/b/c
- 上/下母线电压
- 电池电压、电池电流
- 平衡桥电容电压、平衡桥电容电流
- 模块温度最大值

### 10.2 实时波形

点击“开启实时波形”后：

- 清空已有曲线。
- 按已勾选通道构造 `0x52 0x03` 请求。
- 通过批量阻塞读取接口持续读取数据。
- 图表按块更新，支持 X/Y 轴缩放和拖动。

点击“停止实时波形”后：

- 启用开始按钮。
- 清理通信队列并停止循环读取。

### 10.3 故障录波

功能按钮：

- 读取故障 ID：发送 `0x52 0x01`。
- 读取故障波形：选择故障 ID 后发送 `0x52 0x02`，按偏移分块读取。
- 故障清除：发送 `0x52 0x04 0x01`。
- 清除波形：清空所有曲线并重置坐标轴。

故障索引结构包含：

- 故障 ID
- 记录时间
- 波形总字节数
- 单通道采样点数
- 数据类型，`int16` 或 `float`
- 通道数量
- 周期数量

### 10.4 导入导出

波形导出：

- 保存为 CSV。
- 文件头写入时间戳和通道说明。
- 仅导出当前勾选通道。

波形导入：

- 支持 CSV、TXT 和任意文件过滤。
- 自动识别 `CH` 表头和可选时间列。
- 导入前弹窗展示文件名、通道数、采样点数、是否包含时间数据。
- 确认后替换当前波形并显示导入数据。

## 11. 历史事件

`events` 页面使用扩展功能码 `0x51`。

功能点：

- 读取历史事件数量。
- 每页最多读取 100 条事件。
- 每次请求最多读取 10 条。
- 页面用 50 行、8 列显示，一行展示两条事件。
- 支持上一页、下一页。
- 支持清除历史事件。
- 支持 CSV 导出。
- 支持 PCS 序号选择，选择时写 `40001` 并回读确认。

事件结构：

- `id`：故障 ID。
- `time`：年、月、日、时、分、秒。

故障名称来自 `agreement/故障表` 下 Excel 的 `PCS故障位` 工作表。

## 12. 历史电量

`power` 页面使用扩展功能码 `0x53`。

页面能力：

- 电量类型：充电量、放电量。
- 日期类型：每年电量、每月电量、每日电量、每时电量。
- 点击读取后分 18 次请求，每次读取 10 个点，总计 180 个点。
- 以 30 个点为一页展示。
- 同时展示柱状图和表格。
- 支持上一页、下一页。
- 鼠标悬停柱状图显示日期与电量。

注意：当前页面初始化中存在 `test_power_data()` 模拟数据填充逻辑，真实读取后会用设备响应更新 `powerData`。

## 13. 固件升级

项目包含两个升级入口：`固件升级` 和 `PMU升级`。

### 13.1 PCS 固件升级

`upgrade` 页面支持 PCS 多目标升级，涉及芯片：

- PMU，文件标识 `PA`
- 主芯片 H743，文件标识 `MA`
- 从芯片 F407，文件标识 `SA`
- FPGA，文件标识 `FA`

文件命名约束：

- 注释中约定格式类似 `xxxxMA00001.xxx`。
- 当前代码对 `MA/SA/FA/PA + 5位版本号 + 扩展名` 做校验。
- 对包含 `EPS125P` 的老版本文件有兼容逻辑。

升级流程：

1. 选择需要升级的固件文件。
2. 勾选对应芯片。
3. 点击开始升级。
4. 初始化进度条和状态机。
5. 写升级控制寄存器，使设备进入 boot。
6. 通过 YMODEM 帧发送固件。
7. 读取升级状态和版本信息。
8. 校验升级后版本是否与文件名匹配。

状态机包括：

- `start`
- `INIT`
- `SEND_SOH`
- `SEND_DATA`
- `RETRY`
- `SEND_EOT`
- `ERROR`
- `FAIL`
- `FINISH`

页面还支持“升级压力测试”：

- 主窗口工具菜单可打开/关闭压力测试模式。
- 压力测试 tab 可设置循环次数。
- 支持多固件组合生成规则。
- 记录测量次数、成功次数、失败次数和出错次数。

### 13.2 PMU 分发升级

`PMUpgrade` 页面面向 PMU 通过 SCP 上传固件，并由 PMU 协调 PCS 升级。

主要界面区：

- PCS 信息表：显示 16 台 PCS 的序号、在线状态、PCS-CAN 实际地址。
- 设备状态：PCS 在线台数、升级目标总数、升级结果、升级步骤。
- 操作区：本地固件、远程地址、PCS 选择、文件上传、开始升级、取消升级。
- 日志区。

主要流程：

1. 选择 `.upbin` 文件。
2. 解析文件名识别固件类型 `MA/SA/FA/PA`。
3. 从 `config.json` 的 `PmuUploader` 读取用户名、密码和远端目录。
4. 使用 `SshUploader` 通过 SCP 上传到 PMU。
5. 上传成功后启用“开始升级”。
6. 选择单台 PCS 或“全部”。
7. 写入升级目标总数、升级控制等寄存器。
8. 定时读取升级状态。
9. 升级完成后轮询读取版本并校验。

当前使用的关键寄存器：

- `14001`：PCS 在线台数。
- `40004`：升级目标总数。
- `40005`：升级结果。
- `40007`：当前升级进度和升级步骤。
- `15001 + (PCS序号 - 1) * 500`：PCS 版本信息。
- `15219 + PCS索引 * 500`：PCS-CAN 实际地址相关信息。

## 14. 脉冲调试工具

`WinTest` 页面当前实现一个“脉冲调试”tab。

临时寄存器定义：

## 19. 当前限制与风险

### 19.1 运行路径依赖

`agreement_path` 来自可执行文件目录下的 `agreement`。如果开发环境直接从构建目录运行，而协议目录未复制到可执行文件旁，EMS、历史事件、PMU 升级等功能可能找不到协议或配置文件。

### 19.2 EMS 协议文件选择

EMS 默认取 `agreement/EMS` 目录下第一个文件，不一定是最新版本。目录内存在多个 `V3.00` 到 `V3.13` 文件时，需要确认实际运行时文件排序和选择符合目标设备协议。

### 19.3 通信线程析构风险

`serialThread::~serialThread()` 中直接调用 `tcpSocket->close()`，依赖 `init()` 已创建 `tcpSocket`。正常主窗口启动路径会调用 init，但异常构造/析构路径需注意空指针风险。

### 19.4 模拟数据残留

历史电量页面初始化中调用 `test_power_data()` 填充模拟数据。真实读取后会覆盖，但未读取前 UI 会显示模拟值。

### 19.5 自动化测试入口文档差异

已有自动化测试说明中提到入口位置时存在描述不一致：当前 `mainwindow.cpp` 实现里自动化测试入口位于“工具”菜单。

### 19.6 硬件动作安全

以下功能会直接写控制寄存器，现场使用必须确认设备允许动作：

- 主窗口急停/停止。
- EMS 启动、停止、复位、DC/AC 继电器控制。
- 旧版参数页面的运行控制。
- 固件升级。
- 自动化测试中的启停、轻载、液冷、加热除湿、急停复位。

## 20. 后续维护建议

1. 明确 EMS 协议版本选择规则，避免默认取目录第一个文件造成版本不确定。
2. 将 `agreement/config.json` 的关键配置字段补充成独立配置说明，便于现场维护。
3. 为固件升级和 PMU 升级补充面向操作员的安全确认流程。
4. 历史电量页面移除或显式标识模拟数据，避免误判为设备真实数据。
5. 对 `serialThread` 增加空指针保护和连接关闭状态保护。
6. 对 EMS 动态表格的地址偏移、PACK 配置、flexview 配置补充单元测试或静态校验脚本。
7. 将自动化测试说明中的入口位置与当前实现同步。
