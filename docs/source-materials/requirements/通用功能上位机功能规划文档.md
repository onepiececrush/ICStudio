# 通用功能上位机功能规划文档

> 目标：在当前 EPS520/PCS 调试上位机基础上，升级为一个面向工业设备的“通用调试、监控、仿真、测试、升级一体化上位机平台”。  
> 当前重点：先梳理最终功能边界，方便后续进行 UI 设计、模块划分和架构设计。

---

## 1. 项目定位

本项目最终不只是一个 PCS 调试工具，而是一个通用工业上位机平台。

它应该同时具备以下能力：

1. **主站调试能力**  
   主动读取、写入真实设备数据，例如 PCS、PMU、BMS、液冷、动环、电表、箱变测控等。

2. **从机模拟能力**  
   软件本身可以模拟 PCS、BMS、PMU、液冷、电表等设备，被其他主站、EMS、PLC 或测试程序读取和控制。

3. **自动化测试能力**  
   对通信、参数、控制、故障、升级、联动逻辑进行自动化测试。

4. **协议管理能力**  
   通过 Excel、JSON、CSV 等协议文件生成寄存器表、设备模型、页面和模拟设备。

5. **现场调试能力**  
   支持串口助手、报文收发、实时数据、参数下发、故障查看、波形录波、历史数据和固件升级。

6. **数据分析与组态能力**  
   支持趋势曲线、历史数据、告警、组态大屏和设备拓扑展示。

最终定位可以概括为：

> **工业设备调试、仿真、测试、监控、升级、数据转发一体化平台。**

## 5. 通信中心

通信中心负责所有底层通信链路。

第一阶段建议支持：

- Modbus RTU 主站
- Modbus TCP 主站
- Modbus RTU 从站
- Modbus TCP 从站
- 普通串口收发
- TCP Client
- TCP Server

后续可扩展：

- Modbus ASCII
- CAN / CANopen
- MQTT
- HTTP API
- WebSocket
- OPC UA
- IEC 104
- DLT645 电表协议

### 5.2 主要功能点

- 通信连接配置。
- 多串口通道。
- 多 TCP 通道。
- 多设备地址。
- 发送队列。
- 超时重试。
- 心跳检测。
- CRC 校验。
- MBAP 封装与解析。
- 报文收发日志。
- 报文过滤。
- 报文保存。
- 报文回放。
- 通信质量统计。
- 平均响应时间统计。
- 超时次数统计。
- 异常帧统计。

### 5.3 UI 页面建议

通信中心可以拆成几个 tab：

1. **连接配置**
   - RTU / TCP 模式选择。
   - 串口号、波特率、数据位、停止位、校验位。
   - IP、端口、超时时间、重试次数。
   - 设备地址。
   - 打开 / 关闭连接。

2. **报文助手**
   - 手动输入报文。
   - 十六进制格式化。
   - 发送报文。
   - 接收报文。
   - 清空日志。
   - 保存日志。

3. **报文监控**
   - 实时显示所有请求与响应。
   - 显示时间戳、通道、设备号、功能码、地址、长度、耗时。
   - 支持筛选设备号、功能码、地址范围。

4. **通信统计**
   - 总请求次数。
   - 成功次数。
   - 失败次数。
   - 超时次数。
   - 平均响应时间。
   - 最大响应时间。
   - 最近错误原因。

## 6. 协议管理中心

协议管理中心是通用上位机的核心。

### 6.1 支持的协议来源

- Excel 协议表。
- JSON 协议文件。
- CSV 协议表。
- 手动创建协议。
- 从现有工程导入。
- 从模板库导入。

### 6.2 协议字段

建议协议内部统一抽象为以下字段：

| 字段 | 说明 |
|---|---|
| device_type | 设备类型，例如 PCS、BMS、PMU |
| area | Modbus 区域，例如 holding/input/coil/discrete |
| address | 寄存器地址 |
| name | 点位名称 |
| data_type | 数据类型 |
| word_count | 占用寄存器数量 |
| byte_order | 字节序 |
| scale | 倍率 |
| offset | 偏移 |
| unit | 单位 |
| rw | 读写属性 |
| min | 最小值 |
| max | 最大值 |
| default | 默认值 |
| enum | 枚举值 |
| bit_define | bit 位定义 |
| remark | 备注 |
| group | 分组 |
| page | 所属页面 |
| poll_cycle | 轮询周期 |
| simulate_rule | 模拟规则 |

### 6.3 主要功能点

- 协议导入。
- 协议解析。
- 协议预览。
- 协议字段映射。
- 协议版本管理。
- 协议差异对比。
- 地址冲突检查。
- 数据类型检查。
- 读写属性检查。
- 倍率和单位检查。
- 枚举值管理。
- bit 位解析管理。
- 故障表导入。
- 协议模板导出。
- 自动生成设备模型。
- 自动生成实时监控页面。
- 自动生成参数配置页面。
- 自动生成从机模拟寄存器表。

### 6.4 UI 页面建议

1. **协议列表**
   - 显示当前工程已导入协议。
   - 支持新增、删除、复制、导出。

2. **协议导入向导**
   - 选择 Excel / JSON / CSV。
   - 选择 sheet。
   - 字段映射。
   - 地址格式识别。
   - 数据类型识别。
   - 导入预览。
   - 确认导入。

3. **协议编辑器**
   - 表格方式编辑点位。
   - 支持搜索、过滤、批量修改。
   - 支持按地址排序。
   - 支持按分组查看。

4. **协议校验**
   - 地址重复。
   - 类型缺失。
   - 倍率异常。
   - 读写属性异常。
   - 地址区间冲突。

5. **协议版本对比**
   - 对比两个协议版本。
   - 显示新增、删除、修改的点位。

## 16. 从机模拟中心

这是本项目后续新增的关键模块。

### 16.1 模块目标

从机模拟中心的目标是让本上位机可以模拟各种真实设备，被其他主站、EMS、PLC、网关或测试工具读取和控制。

加上这个模块后，上位机可以同时具备：

```text
当主站：调试真实设备
当从站：模拟设备给别人调试
当测试平台：自动验证协议和功能
当仿真平台：没有硬件也能跑系统
当运维平台：现场查看、配置、升级和分析
```

### 16.2 支持的模拟设备

第一阶段建议支持：

- PCS 模拟。
- PMU 模拟。
- BMS 模拟。
- 液冷模拟。
- 动环模拟。
- 箱变测控模拟。
- 电表模拟。
- 自定义设备模拟。

后续可以扩展：

- 消防模拟。
- 空调模拟。
- UPS 模拟。
- 光伏逆变器模拟。
- 柴油发电机模拟。
- 第三方 EMS 模拟。

### 16.3 支持的从机协议

第一阶段：

- Modbus RTU Slave。
- Modbus TCP Server。

后续：

- Modbus ASCII Slave。
- DLT645 电表模拟。
- CAN 设备模拟。
- MQTT 设备模拟。
- HTTP 接口模拟。

### 16.4 四区数据模型

从机模拟建议支持完整 Modbus 四区：

| 区域 | 功能码 | 说明 |
|---|---|---|
| Coil 线圈 | 01 / 05 / 15 | 可读写布尔量 |
| Discrete Input 离散输入 | 02 | 只读布尔量 |
| Input Register 输入寄存器 | 04 | 只读寄存器 |
| Holding Register 保持寄存器 | 03 / 06 / 16 | 可读写寄存器 |

### 16.5 模拟寄存器字段

| 字段 | 说明 |
|---|---|
| 地址 | Modbus 地址 |
| 名称 | 点位名称 |
| 区域 | coil / discrete / input / holding |
| 数据类型 | uint16 / int16 / uint32 / int32 / float / bit / string |
| 字节序 | ABCD / CDAB / BADC / DCBA |
| 倍率 | 工程值换算倍率 |
| 单位 | V / A / kW / ℃ 等 |
| 默认值 | 启动时的值 |
| 当前值 | 当前模拟值 |
| 最小值 | 允许最小值 |
| 最大值 | 允许最大值 |
| 读写属性 | 只读 / 可写 |
| 变化规则 | 固定 / 随机 / 递增 / 正弦 / 脚本 |
| 关联规则 | 根据其他寄存器计算 |
| 故障规则 | 是否参与故障注入 |

### 16.6 数据变化模拟

支持以下模拟规则：

- 固定值。
- 随机波动。
- 线性递增。
- 线性递减。
- 正弦变化。
- 阶跃变化。
- 斜坡变化。
- 周期变化。
- 根据控制命令变化。
- 根据其他寄存器联动变化。
- 脚本自定义变化。

示例：

```text
电池电压：720V ~ 760V 随机波动
SOC：50% 按 0.1%/分钟递增
PCS 有功功率：收到启动命令后从 0kW 斜坡上升到目标功率
液冷出水温度：围绕 25℃ 小范围波动
```

### 16.7 命令响应逻辑

从机模拟器要能响应主站写入的命令。

示例 1：PCS 启动

```text
主站写入 14051 = 1
→ 模拟 PCS 状态变为“启动中”
→ 2 秒后变为“运行”
→ 有功功率从 0kW 斜坡上升
→ 状态字对应 bit 置位
```

示例 2：PCS 停止

```text
主站写入 14051 = 2
→ 模拟 PCS 状态变为“停机中”
→ 功率逐渐降为 0
→ 状态变为“待机”
```

示例 3：PCS 复位

```text
主站写入 14052 = 1
→ 故障字清零
→ 状态恢复为“待机”
```

### 16.8 故障注入

支持：

- 通信超时。
- 不响应。
- 延迟响应。
- CRC 错误。
- 返回异常码。
- 错误功能码。
- 错误数据长度。
- 错误寄存器值。
- 设备离线。
- 在线 / 离线抖动。
- 电压越限。
- 电流越限。
- 温度越限。
- SOC 异常。
- 故障 bit 置位。
- 启动失败。
- 停止失败。
- 复位失败。
- 液冷故障。
- BMS 告警。
- 动环 DI 告警。

### 16.9 场景脚本

支持用户配置测试场景。

示例：PCS 启动成功

```text
1. 初始状态：PCS 待机，SOC 60%，无故障
2. 收到启动命令
3. 500ms 后状态变为启动中
4. 2s 后状态变为运行
5. 有功功率从 0kW 斜坡升到 50kW
6. 运行 10s 后保持稳定
```

示例：PCS 启动失败

```text
1. 初始状态：PCS 待机
2. 收到启动命令
3. 状态变为启动中
4. 3s 后故障字置位
5. 状态变为故障停机
6. 返回启动失败状态
```

### 16.10 多设备并发模拟

支持：

- 一个串口模拟多个从机地址。

- 多 TCP 端口分别模拟不同设备。
- 多协议实例同时运行。
- 多 PCS 模拟，例如 PCS 1 ~ PCS 16。
- 多 PACK 模拟。
- 多液冷设备模拟。

示例：

```text
COM3：模拟 1~16 号 PCS
COM4：模拟 BMS
TCP 502：模拟 PMU
TCP 1502：模拟电表
```

## 25. 一句话总结

这个通用上位机最终应该做到：

> **一个软件完成真实设备调试、虚拟设备模拟、协议验证、自动化测试、运行监控、故障分析、数据记录和固件升级。**

加上“从机模拟中心”后，它的能力会形成完整闭环：

```text
协议导入
→ 自动生成设备模型
→ 自动生成监控/参数页面
→ 可连接真实设备调试
→ 可模拟虚拟设备给别人调试
→ 可注入故障和异常
→ 可自动化测试
→ 可保存数据和生成报告
```

这就是一个真正强大的通用工业上位机平台。
